# 🎯 Навигация по шагам в Импорт КИЗ v2

## ✨ Что добавлено

Теперь можно **кликать на шаги** в stepper'е для быстрого перехода между этапами импорта!

---

## 🖱️ Как это работает

### Кликабельные шаги

Все **доступные шаги** теперь кликабельные:

```
┌────────────────────────────────────────────────────┐
│  (1) → (2) → (3) → (4) → (5) → (6) → (7)          │
│   ✓     ✓     ✓     ○     ○     ○     ○            │
└────────────────────────────────────────────────────┘
   ↑     ↑     ↑
Можно кликнуть на любой завершенный шаг
```

### Визуальные индикаторы

#### 🟢 Доступный шаг (можно кликнуть):
- **Cursor**: pointer (рука)
- **Opacity**: 1 (полная непрозрачность)
- **Hover**: увеличение (scale 1.1) + зеленая рамка
- **Transition**: плавная анимация

#### 🔴 Недоступный шаг (нельзя кликнуть):
- **Cursor**: not-allowed (запрещено)
- **Opacity**: 0.5 (полупрозрачный)
- **Hover**: без изменений
- **Визуально приглушен**

## 📋 Правила доступности шагов

### Логика определения максимально доступного шага:

```typescript
function getMaxAccessibleStep() {
  if (step === 7) return 7;              // ✅ Импорт завершен - доступны все шаги
  if (uploadedFiles.length === 0) return 1;   // ❌ Файлы не загружены - только шаг 1
  if (!allFilesProcessed) return 1;           // ❌ Файлы обрабатываются - только шаг 1
  if (!selectedProduct) return 2;             // ❌ Товар не выбран - до шага 2
  if (distributions.length === 0) return 4;   // ❌ Нет распределения - до шага 4
  return step;                                 // ✅ Текущий шаг - максимум
}
```

### Примеры:

#### Сценарий 1: Начало работы
```
Текущий шаг: 1
Доступные шаги: [1]
Недоступные шаги: [2, 3, 4, 5, 6, 7]
```

#### Сценарий 2: Файлы загружены
```
Текущий шаг: 2
Доступные шаги: [1, 2]
Недоступные шаги: [3, 4, 5, 6, 7]
```

#### Сценарий 3: Товар выбран
```
Текущий шаг: 3
Доступные шаги: [1, 2, 3]
Недоступные шаги: [4, 5, 6, 7]
```

#### Сценарий 4: Импорт завершен
```
Текущий шаг: 7
Доступные шаги: [1, 2, 3, 4, 5, 6, 7] ✅ ВСЕ!
```

## ⚡ Специальные действия при переходе

При клике на определенные шаги автоматически выполняются необходимые действия:

### Шаг 3 (Подтверждение товара)
```typescript
if (targetStep === 3 && currentStep === 2) {
  autoSelectProduct(); // Автоматически подбирает товар
}
```

### Шаг 5 (Распределение)
```typescript
if (targetStep === 5 && currentStep === 4) {
  initializeDistribution(); // Инициализирует распределение
}
```

/* Обычное состояние */

## 🔄 Способы навигации

Теперь в wizard'е доступно **3 способа** навигации:

### 1. Кнопки в футере
```
[← Назад]                    [Далее →]
```

### 2. Клик по шагам в stepper'е
```
Клик на (1) → Переход к шагу 1
Клик на (2) → Переход к шагу 2
...
```

### 3. Кнопка "На главную"
```
[🏠 На главную] → Выход из wizard'а
```

## 💡 Примеры использования

### Быстрый возврат к шагу 1

Вместо нескольких нажатий "Назад":
```
Шаг 5 → [Назад] → Шаг 4 → [Назад] → Шаг 3 → [Назад] → Шаг 2 → [Назад] → Шаг 1
```

Можно просто кликнуть на шаг 1:
```
Шаг 5 → [Клик на (1)] → Шаг 1 ✨
```

### Проверка данных на предыдущих шагах

Вы на шаге 6 и хотите проверить выбранный товар:
```
Шаг 6 → [Клик на (3)] → Шаг 3 (Подтверждение товара)
       ↓
     Проверили → [Клик на (6)] → Вернулись к шагу 6
```

### После завершения импорта

На шаге 7 (Результат) доступны все шаги - можно посмотреть что было на каждом этапе:
```
Шаг 7 → [Клик на (1)] → Посмотреть загруженные файлы
      → [Клик на (2)] → Посмотреть анализ
      → [Клик на (5)] → Посмотреть распределение
      → [Клик на (7)] → Вернуться к результату
```

## 🛡️ Защита от ошибок

### Нельзя перейти на недоступный шаг

```typescript
// Попытка кликнуть на шаг 5, когда доступен только шаг 3
handleStepClick(5);
// ❌ Переход не произойдет
// Cursor: not-allowed
// Opacity: 0.5
```

### Автоматическая инициализация данных

При переходе на шаги, требующие подготовки данных, они инициализируются автоматически:

```typescript
// Переход на шаг 3
→ autoSelectProduct() выполнится автоматически

// Переход на шаг 5
→ initializeDistribution() выполнится автоматически
```

## 🔧 Техническая реализация

### Основные функции:

```typescript
// Определяет максимальный доступный шаг
getMaxAccessibleStep(): number

// Проверяет, доступен ли шаг для клика
isStepAccessible(stepNum: number): boolean

// Обработчик клика на шаг
handleStepClick(targetStep: number): void
```

### Логика клика:

```typescript
const handleStepClick = (targetStep: number) => {
  const maxStep = getMaxAccessibleStep();
  
  if (targetStep <= maxStep) {
    // Специальные действия при переходе
    if (targetStep === 3 && step === 2) autoSelectProduct();
    if (targetStep === 5 && step === 4) initializeDistribution();
    
    setStep(targetStep);
  }
};
```
