# 🔔 Звуковые уведомления и Toast при загрузке файлов

## 🎵 Обзор

При успешной загрузке и обработке каждого файла система воспроизводит **приятный звуковой сигнал** и показывает **красивое toast-уведомление**.

---

## 🎯 Что добавлено

### 1️⃣ Звуковое оповещение

**Приятный звук успеха** с использованием Web Audio API:

```typescript
playSuccessSound()
```

**Характеристики звука:**
- 🎼 Две ноты: C5 (523.25 Hz) и E5 (659.25 Hz)
- 🎵 Тип: синусоидальная волна (sine)
- ⏱️ Длительность: 0.5 секунды
- 🔊 Громкость: 30% (комфортная)
- 📉 Fade out: плавное затухание

### 2️⃣ Toast уведомление

**Визуальное уведомление** в правом нижнем углу экрана:

```
┌─────────────────────────────────────────────┐
│  ⭕   Отлично!                         ✕    │
│      Файл "document.pdf" успешно обработан  │
└─────────────────────────────────────────────┘
```

```css

    transform: scale(1);       /* Нормальный размер */
  }
  50% {
    transform: scale(1.05);    /* Увеличен на 5% */
  }
}
```

**Параметры:**
- Длительность: 2 секунды
- Повторение: бесконечно (infinite)
- Плавная пульсация иконки

## 🔊 Web Audio API

### Генерация звука

```typescript
const playSuccessSound = () => {
  try {
    // Создаем аудио контекст
    const audioContext = new (window.AudioContext || 
                               (window as any).webkitAudioContext)();
    
    // Создаем два осциллятора для двух нот
    const oscillator1 = audioContext.createOscillator();
    const oscillator2 = audioContext.createOscillator();
    const gainNode = audioContext.createGain();
    
    // Подключаем
    oscillator1.connect(gainNode);
    oscillator2.connect(gainNode);
    gainNode.connect(audioContext.destination);
    
    // Настраиваем частоты (C5 и E5 - мажорное трезвучие)
    oscillator1.frequency.value = 523.25; // C5
    oscillator2.frequency.value = 659.25; // E5
    oscillator1.type = 'sine';
    oscillator2.type = 'sine';
    
    // Настраиваем громкость и затухание
    gainNode.gain.setValueAtTime(0.3, audioContext.currentTime);
    gainNode.gain.exponentialRampToValueAtTime(
      0.01, 
      audioContext.currentTime + 0.5
    );
    
    // Воспроизводим
    oscillator1.start(audioContext.currentTime);
    oscillator2.start(audioContext.currentTime);
    oscillator1.stop(audioContext.currentTime + 0.5);
    oscillator2.stop(audioContext.currentTime + 0.5);
    
  } catch (error) {
    console.log('Audio not supported');
  }
};
```

### Музыкальная теория

**Почему C5 и E5?**
- Это мажорная терция (интервал)
- Звучит радостно и позитивно ✨
- Ассоциируется с успехом и достижением

**Альтернативные варианты:**
- C5 + G5 (квинта) - более "торжественно"
- C5 + C6 (октава) - более "чисто"
- A4 + C#5 + E5 (трезвучие) - более "полно"

## 📱 Toast функционал

### Показать уведомление

```typescript
const showToast = (message: string) => {
  setToast({ message, show: true });
  
  // Автоматически скрыть через 3 секунды
  setTimeout(() => {
    setToast({ message: '', show: false });
  }, 3000);
};
```

### Форматирование имени файла

```typescript
const fileName = fileUpload.file.name.length > 30 
  ? fileUpload.file.name.substring(0, 27) + '...' 
  : fileUpload.file.name;
```

**Логика:**
- Если имя длиннее 30 символов → обрезать до 27 + "..."
- Иначе → показать полное имя
- Предотвращает переполнение toast

## ⏱️ Таймлайн событий

### Процесс загрузки файла:

```
0.0s  → Файл выбран
0.0s  → Начало загрузки (progress 0%)
      ↓ uploading...
2.0s  → Загрузка завершена (progress 100%)
2.5s  → Начало обработки (processing)
      ↓ analyzing...
4.0s  → Обработка завершена (success) ✅
4.0s  → 🔊 Звук успеха
4.0s  → 📢 Toast появляется
      ↓ toast visible...
7.0s  → Toast автоматически скрывается
```

**Итого:** ~7 секунд от выбора до исчезновения уведомления

## 📊 Множественные файлы

### Поведение при загрузке нескольких файлов:

```
Файл 1: загрузка → успех → 🔊 звук → 📢 toast 1
         ↓ 3 секунды ↓
Файл 2: загрузка → успех → 🔊 звук → 📢 toast 2 (заменяет toast 1)
         ↓ 3 секунды ↓
Файл 3: загрузка → успех → 🔊 звук → 📢 toast 3 (заменяет toast 2)
```

**Особенность:**
- Новый toast **заменяет** предыдущий
- Каждый файл воспроизводит свой звук
- Не накапливается стек toast'ов


**2. Успех "колокольчик":**
```typescript
E5 (659.25 Hz) + B5 (987.77 Hz)
Квинта + затухание
```

**3. Успех "монета":**
```typescript
B4 (493.88 Hz) → E5 (659.25 Hz)
Быстрый переход
```
