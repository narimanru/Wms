ТЗ для Figma Make: AI в FASTWMS (UI + Demo)

0) Цель демо

Показать 3 главные ценности AI:
	1.	Проверка поставки до отгрузки (предотвращает ошибки/обезличку/срывы)
	2.	Разбор отклонений УПД (почему отклонили + что делать)
	3.	Подсказки по остаткам/кодам (что докупить и где риск OOS)

⸻

1) Общий стиль
	•	Стиль: как текущий FASTWMS (светлый фон, rounded 16–20, мягкие тени)
	•	AI-акценты: бейдж “AI”, чипы, подсветка рисков, дружелюбный тон (как OpenAI)
	•	Везде короткие тексты, минимум “канцелярита”
	•	Статусы:
	•	✅ OK
	•	⚠️ Риск
	•	❌ Ошибка
	•	Использовать “карточки-результаты” и “рекомендации” (actionable)

⸻

2) Навигация (добавить в сайдбар)

Добавить пункт меню:
	•	AI помощник (иконка ✨)

Внутри AI помощника 3 вкладки:
	•	Проверка поставок
	•	УПД: разбор ошибок
	•	Рекомендации

	•	сверху справа во всём продукте: кнопка “Спросить AI” (открывает чат-оверлей)

⸻

3) Набор экранов (Frames) + что на каждом

Frame A — “AI Hub” (главная AI)

Название: AI Hub / Overview
Цель: один экран = пользователь понимает ценность.

Блоки:
	1.	Hero-карта (слева)

	•	Заголовок: “AI-помощник FASTWMS”
	•	Подзаголовок: “Проверяет поставки, объясняет УПД и подсказывает, что докупить.”
	•	CTA кнопки:
	•	Primary: Проверить поставку
	•	Secondary: Разобрать УПД

	2.	Карточки-сводки (3 шт)

	•	“Риски сегодня” → 4 (кликабельно)
	•	“УПД с ошибками” → 2
	•	“Заканчиваются коды” → SKU 3

	3.	Лента “AI-alerts” (список 5 пунктов)
Пример текста алертов:

	•	“⚠️ SKU: Pants Rib 68906436 — размер M: закончится через 6 дней”
	•	“❌ Поставка #12345: 8 КИЗ отклонены в УПД”
	•	“⚠️ Найдены дубли: 2 кода в партии ‘Март-1’”

⸻

Frame B — “Проверка поставки” (Wizard step 1)

Название: AI / Shipment Check / Step 1
UI:
	•	Заголовок: “Проверка поставки перед отгрузкой”
	•	Выбор поставки (dropdown): Поставка #12345 (WB)
	•	Кнопка: Запустить проверку
	•	Справа: “Что проверяем?” (чипы)
	•	Статусы КИЗ
	•	Дубли
	•	Несоответствие SKU/размер
	•	Не назначены коды
	•	Риски обезлички

⸻

Frame C — “Проверка поставки” (результат)

Название: AI / Shipment Check / Result
Содержание:
	1.	Итоговый скоринг

	•	Большой индикатор: “Риск: Средний”
	•	“Проблем: 3, предупреждений: 5”

	2.	Секция “Критично” (красная)

	•	Карточка 1: “❌ 2 кода уже выбыли из оборота”
	•	Кнопки: Открыть список / Заменить коды
	•	Карточка 2: “❌ 1 код не назначен товару/размеру”
	•	Кнопка: Назначить

	3.	Секция “Предупреждения” (жёлтая)

	•	“⚠️ 5 кодов в статусе ‘не готов к УПД’”
	•	“⚠️ Риск пересорта: размер L → M (по истории операций)”

	4.	Список “Рекомендации AI” (action list)
Каждая рекомендация = 1 действие:

	•	“Заменить 2 выбывших КИЗ на новые из банка кодов”
	•	“Перепечатать 3 этикетки: слабая читаемость (по сканам/жалобам)”
	•	“Перед УПД проверь: 5 кодов должны быть ‘в обороте’”

Важный UX: рядом у каждой рекомендации:
	•	Checkbox “Выполнено”
	•	Кнопка “Перейти” (deep-link на страницу)

⸻

Frame D — “Разбор УПД” (загрузка/выбор)

Название: AI / UPD Analyzer / Select
	•	Заголовок: “Разбор УПД”
	•	Плитки:
	•	“Загрузить по API” (показать как уже подключено)
	•	“Выбрать документ” (list)
	•	Список последних УПД:
	•	УПД #7891 — Обработан с ошибкой
	•	УПД #7890 — Принят

CTA: Разобрать УПД #7891

⸻

Frame E — “Разбор УПД” (результат: понятным языком)

Название: AI / UPD Analyzer / Result
Секции:
	1.	Краткое объяснение

	•	“Проблема: 8 кодов отклонены — 6 из них уже использованы, 2 не в нужном статусе.”

	2.	Таблица “Отклонённые коды” (8 строк)
Колонки:

	•	КИЗ (маска ••••1234)
	•	SKU / Размер
	•	Причина (простым языком)
	•	Рекомендация (кнопка)
Пример причин:
	•	“Код выбыл из оборота”
	•	“Статус кода не подходит для операции”

	3.	План исправления (Step-by-step)

	•	Шаг 1: “Заменить 6 кодов” → кнопка “Открыть банк КИЗ”
	•	Шаг 2: “Обновить назначения” → кнопка
	•	Шаг 3: “Пересоздать/перепровести УПД” → кнопка “Создать заново”

⸻

Frame F — “AI Рекомендации” (остатки/коды/анти-OOS)

Название: AI / Recommendations
3 карточки “Что сделать сегодня”:
	1.	Докупить коды

	•	“SKU Pants Rib / размер M: осталось 22 кода, при темпе продаж хватит на 4–6 дней”
	•	CTA: Купить коды / Создать напоминание

	2.	План поставки

	•	“Для поставки на 300 шт оптимально взять размеры: S 60 / M 120 / L 120”
	•	CTA: Собрать поставку

	3.	Аномалии

	•	“Пересорт по SKU X вырос на 18% за неделю”
	•	CTA: Открыть журнал операций

⸻

Frame G — “Спросить AI” (чат-оверлей)

Название: AI Chat / Overlay
Оверлей справа (как Intercom):
	•	Поле ввода: “Спроси про поставку, коды или остатки…”
	•	Быстрые кнопки (chips):
	•	“Почему УПД отклонён?”
	•	“Что докупить?”
	•	“Где пересорт?”
	•	“Проверить поставку”
	•	Пример диалога (2–3 сообщения):
	•	User: “Почему отклонили 8 кодов?”
	•	AI: “6 кодов уже выбыло из оборота (повторное использование). 2 кода не в нужном статусе. Я выделил их в таблице и предлагаю заменить/обновить назначения.”

⸻

Frame H — “Настройки AI” (минимум)

Название: AI / Settings
	•	Тумблеры:
	•	“Показывать предупреждения перед отгрузкой”
	•	“Автопроверка поставки при создании УПД”
	•	“Еженедельный отчёт в Telegram”
	•	Порог “Риск”: низкий/средний/высокий

⸻

4) Демо-сценарий (Prototype links)

Сделать кликабельный путь на 90 секунд:
	1.	AI Hub → кнопка Проверить поставку
	2.	Step 1 → выбрать Поставка #12345 → Запустить проверку
	3.	Result → клик “Открыть список” (переход на UPD Analyzer / Result или внутренний список)
	4.	Вернуться → AI Hub → Разобрать УПД
	5.	UPD Result → клик “План исправления” → (показать “Банк КИЗ” как stub модалка)
	6.	Открыть AI Chat Overlay → нажать chip “Что докупить?” → ответ AI
	7.	Recommendations показать карточку “Докупить коды”

⸻

5) Данные для моков (чтобы выглядело как настоящий продукт)

Использовать единые примеры:
	•	SKU: Pants Rib, Top Basic, Long Sleeve
	•	Размеры: S / M / L
	•	Поставка: #12345
	•	Метрики:
	•	“Принято: 92 / Отклонено: 8”
	•	“Осталось кодов: 22 (M)”
	•	“Дубли: 2 (критично)”
	•	Маска кода: •••• 4821
