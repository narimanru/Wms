ТЗ для Figma Make

Модуль: Репрайсер FASTWMS

Общая задача

Нужно создать полный UI-макет модуля Репрайсер в системе FASTWMS.
Модуль предназначен для автоматического управления скидками товаров на Wildberries по заданной стратегии.

Нужно отрисовать все основные страницы, состояния, таблицы, модальные окна и боковые панели, связанные с созданием, настройкой, управлением и историей работы репрайсера.

⸻

Важно: стиль интерфейса

Стиль обязателен

Сделать весь интерфейс строго в том же стиле, который уже использовался на странице Products, в едином стиле OpenAI-inspired modern premium SaaS.

Интерфейс должен быть:
	•	минималистичный
	•	светлый
	•	чистый
	•	современный
	•	премиальный по ощущению
	•	без визуального шума
	•	с большим количеством воздуха
	•	с аккуратными белыми карточками
	•	с тонкими серыми границами
	•	с мягкими статусами и чистой типографикой

Важно:

Все экраны должны выглядеть как части одной дизайн-системы FASTWMS, в том же визуальном языке, что и страница Products.
Нельзя смешивать стили.
Нельзя делать один экран в одном стиле, а модалки — в другом.
Все компоненты должны быть консистентными.

⸻

Основной пользовательский сценарий

Пользователь:
	1.	Заходит в модуль Репрайсер
	2.	Создаёт стратегию
	3.	Настраивает параметры
	4.	Добавляет товары
	5.	Настраивает скидки и интервалы времени
	6.	Управляет статусами товаров
	7.	Просматривает историю действий
	8.	Получает уведомления в Telegram
	9.	Контролирует ошибки, изменения WB и ручные изменения

⸻

Поля формы

2. Статус стратегии

	•	выбор подключённого кабинета WB

4. Поле сортировки товаров

	•	Остаток

Блок: Тип стратегии

Карточка выбора типа стратегии.

Текущий тип:

Изменение скидок по времени

Описание:
Автоматическое изменение скидок товаров по расписанию.

Можно визуально зарезервировать место под будущие типы:
	•	По конкурентам
	•	По остаткам
	•	Фиксированная цена

⸻

Блок: Как работает стратегия

Небольшой информационный блок:

Система автоматически меняет скидки товаров по расписанию, учитывает ограничения Wildberries, отслеживает изменения и ведёт историю действий.

	•	список интервалов
	•	кнопка: Добавить интервал

	•	15:00–17:00 → +20%

	•	шаг 30 минут
