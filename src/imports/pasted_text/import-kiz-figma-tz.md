ТЗ для Figma

Модальное окно / Wizard: Импорт КИЗ и привязка к товарам

Цель

Сделать понятный пошаговый интерфейс импорта файлов с кодами маркировки, где система:
	•	принимает файл любого поддерживаемого формата;
	•	автоматически определяет тип файла;
	•	пытается извлечь GTIN и КИЗы;
	•	находит ранее сохраненную привязку GTIN → товар;
	•	при необходимости дает выбрать товар вручную;
	•	позволяет распределить КИЗы между несколькими товарами;
	•	сохраняет привязки для следующих загрузок.

⸻

1. Общая концепция интерфейса

Сделать интерфейс в формате wizard / пошагового мастера.

Формат
	•	модальное окно большого размера или отдельный правый drawer / full modal
	•	стиль в духе современного SaaS-интерфейса
	•	аккуратный, чистый UI
	•	логика должна быть визуально понятной даже без инструкции

Размер

Рекомендуемый размер модального окна:
	•	desktop: примерно 960–1100 px по ширине
	•	высота: адаптивная, с внутренним скроллом при необходимости

⸻

2. Шаги wizard

Шаг 1. Загрузка файла

Задача шага

Пользователь загружает файл с кодами.

Что должно быть на экране
	•	заголовок: Импорт кодов маркировки
	•	подзаголовок:
Загрузите файл, система сама определит формат и попытается извлечь GTIN и коды маркировки
	•	зона drag & drop
	•	кнопка Выбрать файл
	•	поддерживаемые форматы мелким текстом:
PDF, CSV, XLSX и другие поддерживаемые форматы
	•	блок с именем загруженного файла после выбора
	•	индикатор загрузки / обработки файла

Состояния
	1.	Пустое состояние
— файл еще не выбран
	2.	Файл выбран
— видно имя файла, размер, статус обработки
	3.	Ошибка
— файл не читается / формат не поддерживается / файл пустой

Кнопки
	•	Отмена
	•	Далее
активна только после успешной загрузки файла

⸻

Шаг 2. Анализ файла

Задача шага

Показать, что система нашла в файле.

Что должно быть на экране

Карточка результата анализа:
	•	имя файла
	•	тип файла
	•	найден ли GTIN
	•	сколько найдено КИЗов
	•	возможные предупреждения

Блоки

Блок 1. Данные файла
	•	Имя файла
	•	Формат
	•	Дата загрузки

Блок 2. Результат распознавания
	•	GTIN: найден / не найден
	•	Количество КИЗ: N
	•	Статус распознавания:
	•	успешно
	•	частично
	•	есть ошибки

Блок 3. Комментарий системы
Например:
	•	GTIN найден и ранее уже был привязан к товару
	•	GTIN найден, но привязка к товару отсутствует
	•	GTIN в файле не найден, потребуется ручной выбор товара

Кнопки
	•	Назад
	•	Далее

⸻

Шаг 3. Привязка товара

Задача шага

Определить, к какому товару или товарам относятся найденные КИЗы.

Логика

Сценарий 1
Если система нашла ранее сохраненную привязку GTIN → товар:
	•	товар подставляется автоматически
	•	пользователь видит найденный товар
	•	можно подтвердить или заменить

Сценарий 2
Если GTIN найден, но привязки нет:
	•	система просит выбрать товар вручную

Сценарий 3
Если GTIN не найден:
	•	также доступен только ручной выбор товара

⸻

Что должно быть на экране

Верхний информационный блок
	•	GTIN
	•	Найдено КИЗов
	•	Статус привязки:
	•	найдена автоматически
	•	требуется ручной выбор

Основной блок выбора товара
Поля:
	•	поиск товара
	•	выпадающий список / autocomplete
	•	отображение выбранного товара карточкой:
	•	название
	•	артикул
	•	категория
	•	размер / вариант, если есть

Переключатель или бейдж
	•	Найден автоматически
или
	•	Выбран вручную

Чекбокс
	•	Сохранить эту привязку GTIN к товару для следующих загрузок

Если товар выбран вручную, чекбокс по умолчанию включен.

Кнопки
	•	Назад
	•	Далее

Шаг 4. Распределение КИЗ между товарами

Это самый важный экран

Задача шага

Пользователь должен распределить найденные КИЗы между несколькими товарами, если это необходимо.

Логика

После определения основного товара пользователь может:
	•	оставить все КИЗы на одном товаре;
	•	добавить еще товары;
	•	распределить количество КИЗ между несколькими товарами.

⸻

Структура экрана

Верхняя сводка
	•	Всего найдено КИЗов: N
	•	Уже распределено: N
	•	Осталось распределить: N

Таблица распределения
Колонки:
	1.	Товар
	2.	Артикул
	3.	Вариант / размер
	4.	Количество КИЗ
	5.	Действие

Первая строка

Заполняется автоматически выбранным товаром.

Кнопка

+ Добавить товар

При нажатии добавляется новая строка.

⸻

Поведение строк

В каждой строке:
	•	поиск товара
	•	выбор конкретного варианта товара, если нужно
	•	поле количества
	•	иконка удаления строки

⸻

Проверки

Система должна визуально проверять:
	•	сумма по всем строкам = количеству найденных КИЗов
	•	нельзя распределить больше, чем найдено
	•	нельзя оставить 0, если строка активна
	•	нельзя перейти дальше, если остались нераспределенные КИЗы

⸻

Ошибки / алерты

Примеры сообщений:
	•	Количество КИЗ распределено не полностью
	•	Количество КИЗ превышает найденное в файле
	•	Не выбран товар в одной из строк

⸻

Дополнительный UX

Рядом со сводкой сделать цветовой индикатор:
	•	зеленый — распределено полностью
	•	желтый — осталось распределить
	•	красный — ошибка в распределении

Кнопки
	•	Назад
	•	Далее

5. Основная бизнес-логика для разработчика

Это важно учесть в макете текстом/примечанием.

Логика работы
	1.	Пользователь загружает файл
	2.	Система сама определяет тип файла
	3.	Система пытается извлечь GTIN и КИЗы
	4.	Если GTIN уже известен системе, товар подставляется автоматически
	5.	Если GTIN неизвестен, пользователь выбирает товар вручную
	6.	После ручной привязки система сохраняет связь GTIN → товар
	7.	Пользователь может распределить КИЗы между несколькими товарами
	8.	Сумма распределения должна быть равна количеству найденных КИЗов
	9.	После подтверждения данные загружаются в систему

6. Что важно по UX

Интерфейс должен быть:
	•	простым
	•	пошаговым
	•	без перегруза
	•	с понятными подсказками

Нужны подсказки

Возле GTIN, КИЗ и привязки желательно добавить tooltip’ы:
	•	что такое GTIN
	•	зачем сохраняется привязка
	•	зачем нужно распределение

Важно

Пользователь не должен думать, что делать дальше.
Каждый шаг должен буквально вести его за руку.

10. Краткая постановка для Figma designer

Нужно спроектировать пошаговый мастер импорта КИЗ для FASTWMS.
Основная цель — сделать сложный процесс импорта простым и понятным. Пользователь сначала загружает файл, затем система автоматически определяет формат, извлекает GTIN и КИЗы, пытается найти привязку к товару, а затем позволяет распределить КИЗы между одним или несколькими товарами. Интерфейс должен быть максимально понятным, современным и пригодным для дальнейшей разработки.
