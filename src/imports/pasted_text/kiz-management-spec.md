🎯 ТЗ для Figma

Модуль: Управление КИЗами (Банк КИЗов)

⸻

1. Общая концепция экрана

Экран должен отвечать на 3 вопроса:
	1.	Где КИЗ сейчас
	2.	В каком он состоянии
	3.	Что с ним можно сделать

Стиль:
	•	минимализм (в стиле OpenAI / Vuexy)
	•	акцент на данные
	•	много таблиц + быстрые действия
	•	цветовая логика статусов

⸻

2. Структура страницы

Экран: Банк КИЗов

Верхний блок — Сводка (Cards)

Карточки (в 1 ряд):
	•	Доступно
	•	В работе
	•	В производстве
	•	На складе
	•	В поставке
	•	Не принят WB
	•	Под вывод из оборота

👉 каждая карточка:
	•	число
	•	иконка
	•	цвет статуса
	•	кликабельная (фильтр)

⸻

3. Основной блок — Таблица КИЗов

Таблица (главный элемент)

Колонки:
	1.	КИЗ (код)
	2.	GTIN
	3.	Товар
	4.	Размер
	5.	Статус
	6.	Процесс
	7.	Локация
	8.	Поставка / Заказ
	9.	Дата последнего действия
	10.	Действия (иконки)

⸻

Детализация колонок

КИЗ
	•	сокращённый код
	•	копирование по клику

⸻

Статус (цветной бейдж)
	•	Доступен (серый)
	•	В работе (синий)
	•	Наклеен (фиолетовый)
	•	На складе (голубой)
	•	В поставке (оранжевый)
	•	Принят WB (зелёный)
	•	Не принят WB (красный)
	•	Выведен (чёрный)

⸻

Процесс

текст:
	•	Производство / Фабрика 1
	•	Упаковка
	•	Склад
	•	WB
	•	Ozon
	•	и т.д.

⸻

Локация
	•	Склад
	•	Ячейка
	•	BOX-12

⸻

Действия (иконки)
	•	👁 открыть карточку
	•	🖨 печать
	•	🔁 перепечать
	•	📦 добавить в поставку
	•	🏭 в производство
	•	🗑 удалить

⸻

4. Фильтры (обязательно сверху)

Фильтр-бар:
	•	поиск (по КИЗ / GTIN / товару)
	•	статус
	•	процесс
	•	склад
	•	товар
	•	дата

	•	быстрые фильтры:

	•	только проблемы
	•	не принят WB
	•	не привязан
	•	без коробки

⸻

5. Массовые действия (bulk)

При выборе чекбоксов:

Панель сверху:
	•	Привязать
	•	В производство
	•	Печать
	•	В поставку
	•	Переместить
	•	Удалить

⸻

6. Карточка КИЗа (Drawer / Modal)

Открывается справа (drawer)

Блоки внутри:

⸻

1. Основная информация
	•	КИЗ (полный)
	•	GTIN
	•	статус
	•	процесс

⸻

2. Привязка
	•	товар
	•	размер
	•	заказ
	•	поставка

⸻

3. Локация
	•	склад
	•	ячейка
	•	коробка

⸻

4. Действия

Кнопки:
	•	Печать
	•	Перепечать
	•	Переместить
	•	В производство
	•	В поставку
	•	Восстановить (если можно)

⸻

5. Ограничения (ВАЖНО)

Блок:

Если нельзя действие → показать причину:

например:
	•	“КИЗ уже принят WB”
	•	“Нельзя изменить”

⸻

7. История КИЗа

Отдельный блок в карточке

Timeline (вертикальный):

Пример:
	•	Импортирован (дата)
	•	Привязан к товару
	•	Передан в производство
	•	Распечатан (фабрика)
	•	Перепечатан (склад)
	•	Положен в коробку BOX-12
	•	Добавлен в поставку WB-123
	•	Принят WB

⸻

Каждое событие:
	•	действие
	•	дата
	•	пользователь
	•	источник (склад / фабрика)

⸻

8. UX важные моменты

1. Не перегружать
	•	сначала таблица
	•	потом детализация

2. Все действия через кнопки
	•	не через скрытые меню

3. Цвет = смысл
	•	пользователь должен глазами понимать статус

4. Hover / Tooltips
	•	объяснять ограничения

⸻

9. Пустые состояния

Если нет данных:
	•	“Загрузите КИЗы”
	•	кнопка: Импорт

⸻

10. Ошибки / предупреждения

Подсветка строк:
	•	красный → проблема
	•	жёлтый → предупреждение

⸻

11. Дополнительно (если есть время)
	•	закрепление колонок
	•	кастомизация колонок
	•	сохранение фильтров
	•	быстрый экспорт
