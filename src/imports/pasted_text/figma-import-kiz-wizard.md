ТЗ для Figma

Wizard импорта КИЗ / кодов маркировки в FASTWMS

Цель

Спроектировать понятный пошаговый интерфейс импорта КИЗ, где пользователь может:
	•	загружать один или несколько файлов сразу;
	•	видеть процесс загрузки и обработки;
	•	автоматически получать найденный системой товар;
	•	при необходимости удалить автопривязку и выбрать другой товар;
	•	распределять КИЗ между размерами и похожими товарами;
	•	получать предупреждения при рискованных действиях;
	•	видеть подсказки по цвету, размеру, GTIN и соответствию данных файла.

Интерфейс должен быть современным, понятным и подходить для сложного реального сценария селлера / фулфилмента.

⸻

Общая логика сценария

Пример товара

Основной пример в макетах:

Товар: Брюки женские
Цвет: Черный
Размеры: 40, 42, 44, 46, 48, 50

Пример сценария

Пользователь загружает файлы с КИЗами для брюк.
Система анализирует файлы и определяет:
	•	GTIN
	•	размер
	•	цвет
	•	количество КИЗ
	•	возможный товар в системе

После этого пользователь:
	•	подтверждает найденный товар;
	•	либо меняет его;
	•	распределяет КИЗ по размерам;
	•	может добавить еще похожий товар, но система должна предупредить, если это уже другой тип товара или другой цвет.

⸻

Формат интерфейса

Формат окна

	•	ширина desktop: 1100–1280 px

	•	справа основное содержимое шага

Шаги wizard

Цель

Что должно быть на экране
	•	заголовок: Импорт кодов маркировки
	•	подзаголовок:
Загрузите один или несколько файлов. Система сама определит формат, попробует найти GTIN, товар, размер и коды маркировки.

Блок загрузки
	•	drag & drop зона
	•	кнопка Выбрать файлы
	•	подпись:
Можно загружать PDF, CSV и другие поддерживаемые форматы
	•	подпись:
Допускается загрузка нескольких файлов одновременно

⸻

Поведение

Пользователь может загрузить:
	•	1 файл

	•	несколько PDF, где каждый файл соответствует одному размеру;

	•	ошибка

Для больших файлов

Обязательно показать:
	•	прогресс загрузки
	•	прогресс обработки
	•	статус по каждому файлу отдельно

Пример:
	•	Файл 1 — 85% загружено
	•	Файл 2 — анализируется
	•	Файл 3 — найдено 120 КИЗ

Действия у файла

Шаг 2. Анализ файлов и автоматическое определение

Цель

Система анализирует все загруженные файлы и пытается определить:
	•	GTIN
	•	товар
	•	цвет
	•	размер
	•	количество КИЗ
	•	связь с товаром в системе

⸻

Что должно быть на экране

Показать список карточек или таблицу по каждому файлу.

Для каждого файла вывести:
	•	имя файла
	•	найденный GTIN
	•	найденный товар
	•	цвет
	•	размер
	•	количество КИЗ
	•	статус определения

⸻

Варианты определения

Сценарий 1

Система нашла точное соответствие:
	•	товар
	•	цвет
	•	размер

Пример:
Брюки женские / Черный / Размер 48

Статус:
Определено автоматически

⸻

Сценарий 2

Система нашла товар частично:
	•	товар определен
	•	цвет совпадает
	•	размер не уверен

Статус:
Нужно проверить размер

⸻

Сценарий 3

Система нашла GTIN, но нет привязки в системе

Статус:
Требуется ручной выбор

⸻

Сценарий 4

GTIN не найден, система ориентируется только на PDF/название файла

Статус:
Предположительное определение

⸻

Важная логика

Если система автоматически выбрала товар, пользователь должен иметь возможность:
	•	удалить автопривязку;
	•	выбрать другой товар вручную.

Нужна кнопка:
	•	Изменить товар
	•	Сбросить автоподбор

Шаг 3. Подтверждение и выбор товара

Цель

Подтвердить или изменить найденный системой товар.

Что должно быть на экране

Для каждого файла или группы файлов:
	•	карточка найденного товара
	•	кнопка Изменить
	•	кнопка Удалить автоподбор
	•	поиск другого товара

⸻

Пример на брюках

Система нашла:
Брюки женские / Черный / Размер 48

Пользователь может:
	•	оставить это соответствие;
	•	изменить размер;
	•	выбрать другой размер этих же брюк;
	•	выбрать другие брюки;
	•	выбрать вообще другой товар.

⸻

Важные предупреждения

1. Если выбран другой тип товара

Пример:
КИЗы определены как для брюк, а пользователь выбирает футболку.

Показать предупреждение:

Система определила, что КИЗы, вероятно, относятся к товару «Брюки женские».
Вы выбрали товар другого типа — «Футболка женская».
Это может привести к неверной привязке кодов маркировки.
Вы точно хотите продолжить?

Кнопки:
	•	Вернуться и проверить
	•	Продолжить

2. Если выбран другой цвет

Пример:
Система определила:
Брюки женские / Черный

Пользователь выбирает:
Брюки женские / Бежевый

Показать предупреждение:

Система определила цвет «Черный», а вы выбрали товар цвета «Бежевый».
Продолжить можно, но рекомендуется проверить соответствие.

Кнопки:
	•	Проверить
	•	Продолжить

Важно:
не блокировать, только предупредить.

Шаг 4. Проверка размеров и соответствия файлу

Цель

Помочь пользователю не ошибиться с размером.

Это особенно важно при первой загрузке, когда в системе еще нет GTIN-привязки на конкретный размер.

⸻

Логика проверки размера

Если загружен PDF

Система должна попытаться распознать внутри файла:
	•	размер
	•	цвет
	•	модель

Пример:
В PDF найдено:
	•	модель: Брюки женские
	•	размер: 48
	•	цвет: Черный

Если пользователь выбрал размер 50, показать предупреждение:

В загруженном PDF система нашла размер 48, но сейчас выбран размер 50.
Проверьте, пожалуйста, соответствие файла и выбранного размера.

⸻

Если загружен CSV

На первой загрузке, когда GTIN еще не связан с товаром, система не всегда сможет определить размер из содержимого.

Тогда она должна:
	•	посмотреть название файла;
	•	попытаться извлечь подсказки из названия.

Пример названия:
bryuki_black_48.csv

Показать подсказку:

По названию файла предполагается размер 48.
Сейчас выбран размер 50. Проверьте соответствие.

Важно:
	•	это именно подсказка, а не жесткое ограничение.

⸻

Что должно быть на экране

Для каждого файла:
	•	найденный / предполагаемый размер
	•	выбранный пользователем размер
	•	статус совпадения

Статусы:
	•	совпадает
	•	есть расхождение
	•	невозможно определить

Шаг 5. Распределение КИЗ между товарами и размерами

Цель

Распределить найденные КИЗы между размерами или похожими товарами.

КИЗы могут быть:

	•	для нескольких размеров в одном файле;
	•	по одному размеру в каждом отдельном файле;
	•	для похожих товаров одной модели.

⸻

Колонки:

	•	Цвет
	•	Размер
	•	Количество КИЗ
	•	Основание выбора
	•	Действие

Основание выбора

Показывать бейдж:
	•	определено автоматически
	•	выбрано вручную
	•	по GTIN
	•	по PDF
	•	по названию файла

⸻

Кнопки
	•	+ Добавить размер
	•	+ Добавить товар

⸻

Разрешенные сценарии

Можно

Выбрать:
	•	Брюки женские / Черный / 40
	•	Брюки женские / Черный / 42
	•	Брюки женские / Черный / 44
	•	Брюки женские / Черный / 46
	•	Брюки женские / Черный / 48
	•	Брюки женские / Черный / 50

Можно, но с предупреждением

Выбрать:
	•	Брюки женские / Бежевый / 48

Нежелательно, но возможно только после подтверждения

Выбрать:
	•	Футболка женская / Черный / M

⸻

Что должна подсказать система

Если файл относится к брюкам, а выбран другой тип товара

Показать сильное предупреждение:

КИЗы, вероятно, куплены для брюк.
Вы пытаетесь распределить их на другой тип товара.
Рекомендуем перепроверить данные перед сохранением.

Если файл относится к брюкам одного цвета, а пользователь выбирает другой цвет

Показать мягкое предупреждение:

Цвет в файле и выбранный цвет отличаются.
Продолжить можно, но рекомендуется проверить соответствие.

Если размер в файле отличается от выбранного

Показать предупреждение:

Размер, найденный системой, не совпадает с выбранным пользователем.

⸻

Валидации
	•	сумма КИЗ должна совпадать с общим количеством найденных КИЗ
	•	нельзя сохранить пустую строку распределения
	•	нельзя распределить больше найденного количества
	•	нельзя скрыть критичное предупреждение без явного подтверждения пользователем
