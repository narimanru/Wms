Шаг 3 — Подтверждение группы товаров с поддержкой мультицвета

⸻

Цель

Доработать шаг 3 импорта кодов маркировки так, чтобы пользователь мог:
	•	подтвердить автоматически найденный товар
	•	добавить связанные товары
	•	видеть, что группа может быть:
	•	одноцветной
	•	мультицветной
	•	понимать уровень риска при работе с товарами разных цветов

⚠ На этом шаге размеры не выбираются.
Показываем только, какие размеры есть у каждого товара.

⸻

Логика добавления товара

Базовые условия

Товар можно добавить в группу, если совпадает:
	•	категория
	•	связанная модель / группа товаров

Проверка цвета

Если цвет совпадает с основным товаром:
	•	товар добавляется без предупреждения

Если цвет отличается:
	•	товар тоже можно добавить
	•	но система помечает группу как мультицвет

⸻

Состояние группы

После добавления товаров система определяет тип группы:

1. Одноцветная группа

Если у всех товаров одинаковый цвет

Показывать badge:
Одноцветная группа

Цвет badge:
зелёный

⸻

2. Мультицветная группа

Если у товаров разные цвета

Показывать badge:
Мультицветная группа

Цвет badge:
жёлтый / оранжевый

⸻

Предупреждение для мультицвета

Если группа мультицветная, под списком товаров показать alert:

В группе есть товары разных цветов. Проверьте, что распределение кодов допустимо для этих товаров.

⸻

Блок выбранных товаров

После добавления товаров показывается список.

Формат:
карточки или компактная таблица

	•	Фото

	•	Артикул
	•	Цвет
	•	Категория

Одноцветная группа
	•	зелёный badge
	•	без предупреждений

Мультицветная группа
	•	жёлтый badge
	•	alert с предупреждением
	•	можно слегка выделить рамкой блок группы

Что должен понять пользователь

На этом шаге пользователь должен сразу видеть:
	•	какой товар найден
	•	какие ещё товары входят в группу
	•	одноцветная это группа или мультицветная
	•	есть ли предупреждение по цветам

Результат шага

	•	одноцветная
	•	мультицветная

Эти данные передаются на следующий шаг распределения кодов.
