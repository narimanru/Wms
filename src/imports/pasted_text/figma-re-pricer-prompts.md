Отлично, даю тебе готовый набор промптов для Figma Make, чтобы у тебя стал полностью кликабельный прототип репрайсера 👇

⸻

🔥 1. ГЛОБАЛЬНАЯ КОМАНДА (ОБЯЗАТЕЛЬНО СНАЧАЛА)

Вставь это первым:

Добавь интерактивный прототип (Prototype mode) для всего модуля Репрайсер.
Свяжи все экраны между собой через кликабельные элементы.
Используй Open overlay для модальных окон, Navigate для переходов между страницами.
Добавь hover, pressed, active состояния для кнопок, строк таблиц и элементов управления.
Не оставляй статичные экраны без действий.

⸻

🔥 2. СОЗДАТЬ СТРАТЕГИЮ

Кнопка “Создать стратегию” на странице Репрайсер должна вести на экран “Создание стратегии репрайсера — Шаг 1 Настройки”.
Используй Navigate transition.
Добавь hover и pointer cursor.

⸻

🔥 3. КЛИК ПО СТРАТЕГИИ

Сделай строки таблицы стратегий кликабельными.
Клик по строке должен открывать экран редактирования стратегии (Step 1 или Step 2, если уже настроена).
Добавь hover подсветку строки.

⸻

🔥 4. STEP 1 → STEP 2

Кнопка “Создать стратегию” (или “Далее”) на шаге Настройки должна вести на экран “Шаг 2 — Товары”.
Используй Navigate.
Добавь состояние loading при переходе.

⸻

🔥 5. ДОБАВИТЬ ТОВАРЫ (САМОЕ ВАЖНОЕ)

Кнопка “+ Добавить товары” должна открывать modal overlay “Выбор товаров”.
Используй Open Overlay (centered, с затемнением фона).
Модалка должна содержать: поиск, фильтры, список товаров с чекбоксами, кнопки “Добавить выбранные товары” и “Отмена”.
Добавь закрытие по крестику, по кнопке “Отмена” и по клику вне модалки.

⸻

🔥 6. ПОСЛЕ ДОБАВЛЕНИЯ ТОВАРОВ

После нажатия “Добавить выбранные товары” модалка закрывается.
Экран “Шаг 2 — Товары” должен переключиться из empty state в состояние с таблицей товаров.
Создай второй вариант этого экрана (variant) и переключай его через prototype.

⸻

🔥 7. МАССОВЫЕ ДЕЙСТВИЯ

Кнопки “Установить время для всех” должны открывать modal overlay с выбором времени (шаг 30 минут).
После сохранения — обновлять значения в таблице.

⸻

🔥 8. STEP 2 → STEP 3 (ИСТОРИЯ)

Клик по вкладке “История” должен открывать экран “Шаг 3 — История действий”.
Используй Navigate.

⸻

🔥 9. ИСТОРИЯ ПО ТОВАРУ

Кнопка “История” в строке товара должна вести на экран Истории с применённым фильтром по этому товару.

⸻

🔥 10. ДЕТАЛИ СОБЫТИЯ

Клик по записи в истории должен открывать side panel (drawer справа) с деталями: товар, старая цена, новая цена, причина, статус.
Используй Open Overlay (right side drawer).

⸻

🔥 11. TELEGRAM УВЕДОМЛЕНИЯ

Кнопка “Добавить подписку” должна открывать modal “Создать контакт”.
Кнопка “Как подключить” должна открывать вторую модалку с инструкцией.
Добавь переход между модалками.

⸻

🔥 12. КНОПКИ ДЕЙСТВИЙ В ТАБЛИЦЕ

**Иконки действий (редактировать, удалить, дублировать, пауза) должны быть интерактивными:
	•	редактировать → открывает стратегию
	•	дублировать → открывает копию
	•	удалить → подтверждающий modal
	•	пауза → меняет статус inline**

⸻

🔥 13. СОСТОЯНИЯ (ОЧЕНЬ ВАЖНО)

**Добавь состояния:
	•	empty (нет товаров)
	•	loaded (есть товары)
	•	error (ошибка WB)
	•	success (успешно применено)
	•	warning (ограничено WB)

Переключение состояний должно быть через variants или prototype interactions.**

⸻

🔥 14. TIME PICKER

Все выборы времени должны быть только с шагом 30 минут.
Сделай dropdown/time picker с фиксированными значениями (00:00, 00:30, 01:00 и т.д.).
Запрети свободный ввод.

⸻

🔥 15. UX ДЕТАЛИ

**Добавь:
	•	pointer cursor на кликабельные элементы
	•	hover состояния
	•	active состояния
	•	плавные переходы (200–300ms)
	•	затемнение фона при модалках
	•	фиксированные кнопки действий**
