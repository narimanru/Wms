Вот готовое ТЗ для Figma для блока поиска товара на шаге 3 (Подтверждение товара) с таким отображением, как на вашем втором скриншоте.

⸻

ТЗ для Figma

Компонент: Поиск товара (Step 3 — Подтверждение товара)

Цель

При вводе в поиск пользователь должен сразу видеть карточку товара, а не просто текстовый список.

Это позволяет быстро понять:
	•	что это за товар
	•	какой цвет
	•	какой артикул
	•	какая модель

⸻

1. Поле поиска

Label
Выбор товара

Input placeholder:

Поиск товара или артикула…

Поиск работает по:
	•	названию
	•	артикулу WB
	•	артикулу продавца
	•	SKU
	•	категории

⸻

2. Результаты поиска

Результаты отображаются карточками товаров.

Каждая карточка — отдельный selectable item.

⸻

3. Карточка товара

Размер карточки:

height ~80px

layout:

[фото]   Название товара
         Цвета
         Артикул • SKU


⸻

4. Состав карточки

1️⃣ Фото товара

Размер:

60x60 px

Скругление:

8px

⸻

2️⃣ Название товара

Font:

16px
Semibold

Пример:

Брюки палаццо классические широкие с высокой посадкой

⸻

3️⃣ Цвета товара

Под названием

Font:

14px
Color:

#6B7280

Пример:

бордовый, сливовый, красновато-бордовый, бургунди

Если цветов больше 4:

бордовый, сливовый, красновато-бордовый +3


⸻

4️⃣ Артикулы

Нижняя строка

Font:

14px

Цвет:

#9CA3AF

Формат:

529940427 • Bruki-bez-zashipov-bordo

где

первое — артикул WB
второе — артикул продавца

⸻

5. Hover состояние

При наведении:
	•	светлый фон
	•	рамка

background: #F9FAFB
border: #E5E7EB


⸻

6. Selected состояние

При выборе товара:
	•	зеленая рамка
	•	галочка

border: #22C55E


⸻

7. Если товар найден автоматически

Справа badge:

Найден автоматически

цвет:

#DCFCE7


⸻

8. Если несколько цветов

Показывать предупреждение:

badge

Мультицвет

tooltip:

У товара несколько цветов.
Проверьте соответствие кодов маркировки.


⸻

9. После выбора

Карточка перемещается в блок:

Выбранные товары

И отображается как:

[фото]  Брюки палаццо классические
       бордовый
       529940427


⸻

10. UX поведение

При поиске:
	•	показывать до 10 товаров
	•	сортировка:
	•	точное совпадение
	•	частичное совпадение
	•	популярные

⸻

11. Empty state

Если ничего не найдено:

Товар не найден

Попробуйте:
• изменить название
• искать по артикулу
• выбрать товар вручную

Кнопка:

Открыть каталог товаров

⸻

12. Mobile версия (если будет)

Карточки:
	•	фото слева
	•	текст справа
	•	высота ~72px

⸻

Итог

После доработки поиск будет показывать:

✔ фото
✔ название
✔ цвета
✔ артикул WB
✔ артикул продавца

и пользователь сразу понимает какой товар выбирает.

⸻

Если хочешь, я ещё покажу очень важную UX-фичу для этого шага, которая сильно улучшит импорт КИЗов в FastWMS (и её нет ни в одной системе для WB).
