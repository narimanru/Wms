Дополнение к ТЗ: AI “Автопилот” (One-click execution)

0) Идея

Автопилот = режим, где пользователь:
	1.	формирует план (30 дней / сезон),
	2.	нажимает “Запустить автопилот”,
	3.	выбирает, что именно создавать (заказы/поставки/закупки КИЗ/задачи),
	4.	подтверждает,
	5.	FASTWMS создаёт “пакет действий” + показывает прогресс и результат.

Важно: автопилот не делает критические действия без подтверждения, только создаёт задачи/черновики/заказы.

⸻

1) Где включается

На AI Planner / Dashboard (30 days) и AI Planner / Dashboard (Season) добавить кнопку:
	•	Primary: Запустить автопилот
	•	Вторичная рядом: “Режим: Черновик” (dropdown)
	•	Черновик (безопасно) — создаёт черновики поставок/заказов
	•	Полуавто — создаёт и назначает ответственных + дедлайны
	•	Авто (future) — скрыть/заблокировать в демо или пометить “скоро”

⸻

4) Прототип-поток (1–2 минуты)
	1.	Dashboard (Season) → Запустить автопилот
	2.	Autopilot Setup → Создать пакет действий
	3.	Autopilot Review → клик “Добавить к покупке КИЗ” (галочка решено) → Запустить
	4.	Running → Открыть результат
	5.	Result → клик “Открыть задачи” → Created Objects таб “Задачи”
	6.	Из “Поставки” клик на Draft → Shipments Plan (скролл к нужным строкам)

⸻

5) Визуальные “вау” детали (обязательно)
	•	На Running сделать ощущение автоматизации: чекмарки появляются по шагам (в прототипе — переходы/варианты).
	•	На Review конфликт-карточки выглядят как “AI нашёл проблему и сразу предложил кнопку решения”.
	•	Везде писать “черновик / draft”, чтобы не пугать автодействиями.
