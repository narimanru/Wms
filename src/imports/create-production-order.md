Ниже — ТЗ для Figma Make на страницу “Создать заказ на производство одежды” в FASTWMS. Я сделаю так, чтобы это работало и для селлера (свой цех/подрядчик), и для фулфилмента/бренда, и чтобы легко связалось с AI-планировщиком, КИЗ и отгрузками WB.

⸻

ТЗ для Figma Make: Страница “Создать заказ на производство”

0) Цель страницы

Быстро создать производственный заказ (PO/MO) с:
	•	выбором товаров/размеров и плановыми количествами,
	•	материалами/фурнитурой/операциями,
	•	маркировкой (КИЗ) и упаковкой,
	•	сроками, статусами и ответственными,
	•	расчётом себестоимости и предоплаты,
	•	связью с планом отгрузок на WB (опционально).

⸻

1) Размещение в навигации

Sidebar:
	•	Производство
	•	Заказы
	•	Поставщики/Фабрики
	•	Материалы (опционально)
	•	Техкарты (опционально)

Кнопка в “Заказы”:
	•	+ Создать заказ

⸻

2) Общий стиль
	•	Как FASTWMS: light UI, rounded 16–20, таблицы как в продукте.
	•	Страница — wizard (шаги) + возможность “сохранить черновик”.
	•	Везде подсказки (tooltips) и автозаполнение.
	•	Статусы: Draft → Approved → In production → QC → Ready → Shipped → Received.

⸻

3) Структура: Wizard из 5 шагов

Верх страницы: Stepper (1–5) + “Сохранить черновик”.

Шаг 1. Основное

Frame: Production Order / Create / Step 1 - Basics

Поля
	•	Номер заказа (авто): PO-2026-00041
	•	Фабрика / Подрядчик (dropdown) + кнопка “+ добавить”
	•	Организация / Бренд (если мульти-орг)
	•	Сезон / Коллекция (dropdown): Весна 2026
	•	Тип заказа (chips):
	•	Производство (пошив)
	•	Дозаказ
	•	Перемаркировка (опц.)
	•	Плановая дата готовности (date)
	•	Дата старта (date)
	•	Условия оплаты:
	•	Предоплата % / сумма
	•	Остаток по готовности/отгрузке
	•	Валюта (RUB/THB/USD — опц.)

Блок “Связь с планом” (если AI планировщик)
	•	Toggle: “Связать с AI-планом” (ON)
	•	Dropdown: “План: Весна 2026 / 30 дней”
	•	Чип: “Причина: пополнение WB Краснодар / дефицит размеров M/L”

CTA
	•	Primary: Дальше
	•	Secondary: “Сохранить черновик”

⸻

Шаг 2. Состав заказа (SKU/размеры/кол-ва)

Frame: Production Order / Create / Step 2 - Items

Блок “Добавить товары”
	•	Search: “Найти товар / SKU”
	•	Кнопки:
	•	Добавить вручную
	•	Импорт из Excel/CSV
	•	Из AI-рекомендаций (если включен планировщик)

Таблица состава заказа

Колонки:
	•	SKU / Наименование
	•	Цвет (если есть)
	•	Размер
	•	Кол-во
	•	Цена за пошив (план)
	•	Сумма
	•	КИЗ нужно (qty)
	•	Статус КИЗ (достаточно/дефицит)
	•	Комментарий

UX-фишки
	•	Группировка по SKU (с раскрытием размеров)
	•	Быстрые действия:
	•	“Распределить по размерам” (modal)
	•	“Округлить до мин. партии” (если правило)
	•	Под таблицей итоги:
	•	Итого шт
	•	Итого сумма
	•	Итого КИЗ нужно

Side panel “AI подсказки” (необяз. но красиво)
	•	“Размер M самый дефицитный на WB — рекомендовано +120”
	•	“КИЗ дефицит 25 — добавьте в план КИЗ”

⸻

Шаг 3. Материалы, фурнитура, упаковка

Frame: Production Order / Create / Step 3 - Materials

3 вкладки (tabs)
	1.	Материалы
	2.	Фурнитура
	3.	Упаковка/этикетки

Материалы (таблица)
Колонки:
	•	Материал (ткань)
	•	Артикул поставщика
	•	Норма расхода (м/шт)
	•	Нужно (итого)
	•	На складе
	•	Дефицит
	•	Поставщик
	•	Дата поступления (ETA)

Кнопки:
	•	Создать заявку на закупку (если дефицит)
	•	Резервировать со склада (опц.)

Упаковка/этикетки
Чек-боксы:
	•	Пакет/зип-лок
	•	Стикер размера
	•	Бирка
	•	Инструкция/вкладыш

Маркировка:
	•	Toggle: “Требуется Честный знак” (ON)
	•	“КИЗ печать”:
	•	Печатает: Фабрика / Мы / Фулфилмент
	•	Формат: PDF / CSV
	•	Требования к этикетке (tooltip)

⸻

Шаг 4. План работ (операции) и контроль качества

Frame: Production Order / Create / Step 4 - Workflow & QC

Timeline / чек-лист операций

Список этапов с датами и ответственными:
	•	Крой
	•	Пошив
	•	ВТО
	•	ОТК
	•	Упаковка
	•	Маркировка/наклейка
	•	Отгрузка

Для каждого:
	•	Дата начала/окончания
	•	Ответственный (dropdown)
	•	SLA/норма (опционально)

QC чек-лист (карточки)
	•	Размерная сетка соблюдена
	•	Швы/дефекты
	•	Бирки/упаковка
	•	Маркировка наклеена корректно
	•	Скан DataMatrix проходит

⸻

Шаг 5. Итоги и создание

Frame: Production Order / Create / Step 5 - Review

Summary cards (4 шт)
	•	Заказ: PO-2026-00041
	•	Итого: 1 050 шт / 1 980 000 ₽
	•	Готовность: 25 марта
	•	КИЗ: нужно 1 050 / дефицит 25

Блок “Риски” (если есть)
	•	❌ “Не хватает КИЗ: 25” → кнопка Добавить в план КИЗ
	•	⚠️ “Материал X дефицит 120 м” → Создать закупку
	•	⚠️ “Сроки: риск опоздания на 3 дня” → Сдвинуть/разбить партию

CTA
	•	Primary: Создать заказ
	•	Secondary: “Сохранить как черновик”
	•	Link: “Печать спецификации PDF” (в демо просто кнопка)

⸻

4) После создания — экран заказа (Detail)

Frame: Production Order / Detail

Header
	•	PO-2026-00041 — Status badge: Draft/Approved/In production…
	•	Buttons:
	•	Утвердить
	•	Отправить фабрике
	•	Добавить оплату
	•	Создать приемку
	•	Создать отгрузку на WB (если связка)

Вкладки (tabs)
	1.	Состав (SKU/размеры)
	2.	Материалы
	3.	Статусы работ
	4.	Финансы
	5.	Документы (договор/счёт/акт)
	6.	Маркировка (КИЗ)

⸻

5) Модалки (обязательные)
	1.	Распределить по размерам
Ввод: общий объём + проценты/шт по S/M/L → применить.
	2.	Импорт из Excel/CSV
Маппинг колонок + preview.
	3.	Добавить фабрику
Название, контакты, сроки, реквизиты.
	4.	Добавить оплату
Сумма, дата, метод, комментарий.
	5.	Создать заявку на закупку материалов (если дефицит)

⸻

6) Демо-данные (для прототипа)
	•	Фабрика: “Kyrgyz Sewing #1”
	•	Заказ: PO-2026-00041
	•	SKU:
	•	Pants Rib: S 300, M 400, L 300
	•	Top Basic: S 200, M 180, L 150
	•	КИЗ дефицит: Long Sleeve M 25 (пример)
	•	Материал “Rib cotton”: нужно 980 м, на складе 760 м, дефицит 220 м

⸻

7) Прототип-сценарий (1–2 минуты)
	1.	List → Создать заказ
	2.	Step 1 заполнить фабрику/сезон → Next
	3.	Step 2 → “Из AI-рекомендаций” → добавились строки → Next
	4.	Step 3 → показ дефицита материала → “Создать заявку” (toast)
	5.	Step 5 → видим риски “КИЗ дефицит 25” → “Добавить в план КИЗ”
	6.	Создать заказ → перейти на Production Order / Detail

⸻

8) Микротексты (короткие)
	•	“Заказ на производство — это план выпуска по размерам + сроки + маркировка.”
	•	“FASTWMS проверит КИЗ и материалы и покажет риски до запуска.”

⸻

Если хочешь, следующим шагом могу написать ТЗ на список заказов производства (таблица/фильтры/статусы/канбан), чтобы “создать заказ” не висел в воздухе, а был частью полного модуля.
